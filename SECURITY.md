# Security Policy

## Supported Versions

The latest version on `main` is the primary supported version.

## Reporting A Vulnerability

Please do not disclose security issues publicly in GitHub issues.

If you discover a vulnerability, especially one involving:

- local credential storage
- API key leakage
- unsafe request execution
- privilege escalation
- unintended network exposure

please report it privately to the maintainers first.

Until a dedicated security contact is published, open a private GitHub security advisory if available for the repository owner, or contact the maintainer directly through a private channel.

## What To Include

- Clear description of the vulnerability
- Steps to reproduce
- Impact assessment
- Suggested mitigation if known

We will aim to acknowledge valid reports quickly and coordinate a responsible fix before public disclosure.
